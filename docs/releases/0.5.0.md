# FreshSense AI 0.5.0 Public Beta

FreshSense 0.5.0 is the first public Windows beta intended for nontechnical
testers. It packages the desktop application, DenseNet201 model,
supported-input gate, local semantic retrieval assets, reviewed knowledge base,
and metadata-only scan history into one per-user installer.

## Download

Download all three release files from the GitHub Release:

- `FreshSenseAI-Setup-0.5.0.exe`
- `FreshSenseAI-Setup-0.5.0.exe.sha256`
- `FreshSenseAI-Release-0.5.0.json`

Verify the installer before running it:

```powershell
Get-FileHash .\FreshSenseAI-Setup-0.5.0.exe -Algorithm SHA256
```

Compare the hash with the `.sha256` file or release manifest.

## What is new

- Public-beta versioning and release metadata.
- Redesigned Streamlit scanner with bundled fruit samples and progressive
  disclosure for Grad-CAM, retrieval evidence, diagnostics, and agent trace.
- A privacy-conscious Report incorrect result action in desktop and Streamlit.
- GitHub issue templates for prediction and usability feedback.
- CSV import for structured, metadata-only pilot observations.
- A reproducible orange failure-review manifest, prediction-distribution report,
  and Grad-CAM export workflow.
- Clearer README download, privacy, supported-input, and limitation guidance.

## Privacy

FreshSense analyzes selected photos locally. It does not create an application
copy or automatically upload a photo. Scan history contains metadata only. A
tester may voluntarily attach a de-identified test photo to a GitHub issue.

## Safety and known limitations

- FreshSense provides visual decision support only. It cannot determine whether
  food is safe to eat or detect internal spoilage, contamination, odor, texture,
  pathogens, or chemical hazards.
- Use one clear apple, banana, or orange fruit type per photo.
- The current model lacks an independent real-world accuracy estimate. Legacy
  data contains source-group leakage, so earlier 97 to 99 percent results are
  not valid real-world accuracy claims.
- The synthetic unsupported false-acceptance rate is 5.73 percent and is not a
  substitute for real unsupported photographs.
- More than 50 users were reported in earlier informal testing, including three
  orange errors, but case-level data and post-change retesting are incomplete.
- This beta is not Authenticode signed. Windows may show an unrecognized
  publisher warning. The SHA-256 checksum verifies file integrity but not
  publisher identity.
